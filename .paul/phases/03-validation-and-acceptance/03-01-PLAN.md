```gherkin

```

## AC-2: Validation Summary is Structured and Traceable
```gherkin
Given validation command execution results
When runner returns
Then output includes per-command status, exit code, and concise diagnostics
```

## AC-3: Pipeline Enforces Validation Gating
```gherkin
Given selected mode is apply and validation fails
When pipeline finalizes orchestration output
Then final mode is forced to propose and remediation appears in next action/summary
```

## AC-4: Pipeline Keeps Writes Safe on Validation Failure
```gherkin
Given validation fails
When artifact materialization decision is made
Then no apply writes occur and output remains propose-oriented
```

## AC-5: Validation Integration Tests
```gherkin

When orchestrator tests execute
Then tests verify pass/fail gating behavior and output contract fields
```
