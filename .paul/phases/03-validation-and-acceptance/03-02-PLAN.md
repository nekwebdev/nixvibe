<acceptance_criteria>

## AC-1: Init Journey Acceptance
```gherkin
Given a natural-language scaffold request in a greenfield context
When orchestration runs
Then route resolves to init
And mode behavior is explicit and safe
And core init artifacts are present with one immediate next action
```

## AC-2: Audit Journey Acceptance
```gherkin
Given a natural-language refactor request in an existing config context
When orchestration runs
Then route resolves to audit
And default mode is propose
And audit artifacts include plan, target tree, and patch proposal paths
```

## AC-3: Apply Safety Acceptance
```gherkin
Given apply intent with explicit opt-in
When validation passes
Then writes are allowed
And when validation fails, final mode is forced to propose with no writes
```

## AC-4: Output Contract Acceptance
```gherkin
Given any accepted orchestration result
When user-facing output is inspected
Then selected mode, artifact summary, and next action are always present
```

</acceptance_criteria>
