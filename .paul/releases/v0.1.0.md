# Release Notes: v0.1.0

Released: 2026-04-19

## Summary

v0.1.0 ships the first end-to-end nixvibe orchestration baseline:
- deterministic route/mode/conflict policy behavior
- structured parallel specialist execution and merge pipeline
- scaffold/refactor artifact generation with safe write-mode handling
- mandatory validation gates before apply writes
- acceptance and patch hygiene regression coverage

## Highlights

1. Policy and contracts
- CARL policy contract locked at `.agents/carl/nixvibe-domain.md`
- Local MCP contracts committed for Codex and Claude
- Specialist structured output schema locked and validated

2. Runtime orchestration
- Route selection (`init` / `audit`) implemented and tested
- Confirm-first write modes (`advice`, `propose`, `apply`) enforced
- Deterministic conflict resolution by policy priority
- Parallel specialist runners with typed payload validation

3. Artifact and validation pipeline
- Dendritic scaffold and audit artifact generation
- Mode-gated materialization (no surprise writes)
- Apply path validation gate with `nix flake check` and `nix fmt`
- Validation failure safely downgrades apply to propose

4. Acceptance and hygiene
- End-to-end journey tests for init/audit/apply safety
- Patch path normalization to safe `patches/*.patch`
- Release check script: `scripts/release-check.sh`
